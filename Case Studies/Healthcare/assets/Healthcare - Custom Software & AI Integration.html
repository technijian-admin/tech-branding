The Challenge

A medical practice ran a live web application supporting day-to-day clinical and authorization workflows. The user interface was aging, the database schema needed to evolve to support new data requirements, and clinical staff were spending too much time manually assembling authorization checklists — a task that was repetitive, error-prone, and a drag on patient throughput.

The practice also wanted to explore AI assistance for the checklist task, but without risking data-privacy violations or shipping something untested into a clinical workflow. They needed a partner who could do the software modernization and the AI integration inside a single engagement with HIPAA-aware controls.

  • Aging UI on a live healthcare web application
  • Database schema limits blocking new data-capture requirements
  • Manual authorization checklists — repetitive, error-prone, slow
  • No programmatic data service between UI and downstream systems
  • AI aspiration without AI governance — interest in OpenAI, zero tooling in place
  • HIPAA posture must be preserved through every change
The Solution

Technijian delivered the work in a three-track development engagement: UI + database modernization of the live web application, an MCP-compatible data service stood up to feed the UI, and an OpenAI Assistant integrated to auto-generate authorization checklists formatted to UI specifications.

Track 1
UI & Database Update
Iterative UI redesign for accessibility with user feedback loops; database schema updates to support new data requirements; weekly stakeholder check-ins and change logs.
Track 2
MCP Data Service
Stood up an MCP-compatible server to facilitate clean data access between UI and database, with monitoring for uptime and performance metrics.
Track 3
OpenAI Assistant
Integrated an OpenAI Assistant to generate authorization checklists formatted to UI specifications, with defined input data formats and data-privacy compliance.
