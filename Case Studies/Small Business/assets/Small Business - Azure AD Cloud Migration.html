The Challenge

A 21-user small business relied on an aging on-premise Active Directory server for user logins, file access, and group policy. The server was past its useful life, the cost of a physical refresh was hard to justify for the user count, and remote work had made the on-prem domain model feel increasingly out of step with how the team actually worked.

The business wanted to move identity to Microsoft Entra ID (Azure AD), decommission the on-prem server, and get out of the hardware-refresh cycle — without breaking OneDrive, Outlook, or the user profiles people had built up over years.

  • Aging on-prem AD with 21 user accounts and local profiles
  • Hardware refresh looming — hard to justify CapEx for 21 users
  • Remote work made the on-prem model increasingly awkward
  • OneDrive & Outlook must survive the identity cutover intact
  • User profiles need to transfer, not be rebuilt
  • No time for an extended outage — people have work to do
The Solution

Technijian planned and executed a phased cloud identity migration: full environment assessment, target-state design, controlled cutover of each desktop, validation of OneDrive and Outlook functionality post-cutover, and formal decommissioning of the legacy on-prem server with documentation.

Phase 1
Evaluation
Assess current AD environment, identify needs and risks, produce an evaluation report that guides the migration strategy.
Phase 2
Design
Design the target Entra ID (Azure AD) environment: user roles, access levels, and policies documented before any change.
Phase 3
Implementation
Unjoin each desktop from the on-prem domain, join Entra ID, transfer user profiles, validate OneDrive and Outlook.
Phase 4
Decommission
Verify all data preserved in cloud, back up and securely decommission the old on-prem AD server, document the new setup.
What We Preserved
  • User identities — all 21 users transitioned cleanly to Entra ID
  • OneDrive — file sync and share behavior validated post-migration
  • Outlook — mail, calendar, contacts intact through identity change
  • User profiles — desktop profile transfer, not forced rebuild
  • Access levels — role/permission mapping preserved in the new design
  • Business continuity — staggered cutovers, not a big-bang weekend

The Results

The business now runs identity entirely in Microsoft Entra ID. The old on-prem server has been securely decommissioned. OneDrive, Outlook, and user profiles survived the cutover. The next server refresh doesn’t exist — because the server doesn’t exist anymore. And the whole environment is documented for the next IT engagement, whether it’s Technijian or not.
