Case Study · AI Development — Chat.AI Platform & GSD Pipeline
Why the Pipeline Works in Production

Most autonomous dev systems don’t survive first contact with a real codebase. GSD does, because four specific decisions are baked in from day one:

  • Vault-backed memory. Every agent’s system prompt, every architecture contract, every orchestrator decision lives in an Obsidian vault indexed by the agents themselves. Humans edit prompts in the vault, not in code — runtime picks up the change on next run.
  • Dual-auth routing for $0 marginal cost. Primary path uses CLI OAuth subscriptions (Claude Code, OpenAI Codex, Gemini). API keys are a backup path that only triggers on rate limit. Steady-state cost stays at the fixed subscription price.
  • Impact-analysis gates. No agent renames a symbol without running a call-graph impact analysis first. HIGH / CRITICAL risk halts for human review. No blind refactors.
  • Three-model code review. Every review cycle runs Claude, Codex, and Gemini in parallel against the same diff. Findings merged; RemediationAgent picks up the union set.
