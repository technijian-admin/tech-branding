An hourly, idempotent bridge between DaisyBill and ORX — billing transactions, payments, submissions & errors, ready for Claude and reporting.
The Exchange — How Data Moves
1
n8n scheduler — triggers MCP hourly via POST /jobs/sync on daisybill.mcp.orthoxpress.com
2
MCP pulls DaisyBill — bearer-token calls to api.daisybill.com/v1 with updated_at watermark + 2h overlap
3
Fan-out per bill — detail, payments, submissions, attachments, tasks, notes & error reports
4
Hand-off to DB API — MCP never touches SQL; posts upsert package to daisybill.api.orthoxpress.com
5
SP-only persistence — DB API invokes stored procedures; raw JSON archived in dbo.RawApiPayload
6
Claude consumes via MCP — stdio or HTTP transport over protocol 2024-11-05: tools, resources & prompts

Capabilities at a Glance
Incremental, Idempotent Sync
  • Watermarked updated_at pulls with overlap window
  • UPSERT on natural keys — safe to replay any run
  • Exponential backoff + jitter on 429/5xx; circuit-break on outage
Raw + Normalized Persistence
  • Every upstream payload archived for audit & replay
  • Normalized domain tables drive reporting & the admin portal
  • PHI-aware: bodies redacted from logs, encrypted at rest & in transit
Observability & Health
  • GET /healthz liveness + readiness on MCP and DB API
  • Structured logs with correlation IDs per sync run & bill
  • Dashboard stats, per-provider sync success, row counts & latency
