My Jian AI Agent — What It Does & Monitors
V4 · Effective 2026-06-15 · Functional Reference • all times Pacific
How It Works
The Four Functions & Cadence
  • Alert — real-time critical-event detection. Every 15 min (lower tiers 30m–6h) → opens a P1/P2 Client-Portal ticket + Teams channel + Adaptive Card.
  • Diagnose — the AI judgment pass: reads the day’s data, decides what warrants a ticket + recommended fix + MITRE tag. Daily 14:00 (or at Alert cadence on critical platforms).
  • Activity — daily roll-up into a per-client posture summary. Daily 14:30 → feeds monthly reports, the SIEM correlator and the learning loop.
  • Counts — usage/license reconciliation (observed vs. billed) to surface under-billing & license waste. Observed daily 05:00; billed monthly.
Compliance Engine  31 evaluators • 5 frameworks
  • Frameworks: HIPAA, SOC 2, CMMC, PCI-DSS + Technijian baseline. Compliance tier adds SOX / GLBA / FINRA / NIST 800-171 / FedRAMP.
  • Alert mode — 6 critical-breach evaluators every 15 min (MFA disabled, audit log off, CA policy disabled, DMARC reverted, new mailbox forwarding rule, new Global Admin).
  • Diagnose — all 31 evaluators daily 03:00. Activity — pass/fail by framework daily 14:30.
  • Produces: framework-tagged evidence, a branded monthly compliance report (DOCX), and an append-only tamper-evident audit log. Evaluators return “unknown” when they can’t prove a result — Jian never claims a control passes without evidence.
SIEM / Correlation  858 MITRE techniques
  • MITRE ATT&CK: 858 techniques • 15 tactics • 189 threat groups • 824 software, refreshed daily 02:00 from the official MITRE STIX bundle.
  • Real-time every 15 min (1-hour window) → P1 the moment a critical correlation crosses threshold. Retrospective daily 14:30 re-scans the prior 7 days against the fresh catalog (catches slow-burn campaigns).
  • Correlates 5 sources: Inky, Sophos, M365 Security (sign-in), CrowdStrike, Huntress — kill-chain scored by how many distinct tactics fired.
Inky phish→ M365 risky sign-in→ CrowdStrike PowerShell→ Huntress task→ CRITICAL coordinated-attack ticket
How Jian Communicates
  • Email (MS Graph, from jian@technijian.com) — daily Morning Summary (07:00), rich-HTML security alerts (MITRE map + IP-geo table + AADSTS codes in plain English + what Jian did + next steps), user-direct notices, weekly autonomy report (Mon 07:00).
  • Microsoft Teams — the @Jian bot — channel-per-ticket (auto-creates, renames [CLOSED] + archives on close), Adaptive Cards, a dedicated “Jian Alerts” channel, two-way chat, and a self-service assistant on a per-client RAG brain. Live in 99 of 100 client Teams tenants.
  • Client Portal tickets — idempotent (never double-files), forensic context, shift-aware routing (India-primary / US-fallback, L2/L3 escalation), automatic billable time entries.
Remediation is gated by design. Jian opens tickets — it does not silently fix client systems. The only live autonomous actions are Jian self-healing its own platform and M365 session revocation on confirmed account takeover (plus a small fixed catalog of reversible infra actions). Every client-platform Tier-1 fix requires 7 days of clean diagnose traces matching what techs did (≥95%), explicit per-op approval, and a tested rollback path; the first (Sophos firewall) is roadmapped 2026 Q2. Jian shows you a documented audit trail proving it would have made the right call — N times — before it is ever empowered to act. (It emails the malicious-IP list for manual block; Microsoft blocks programmatic IP-blocking, so only session-revoke is automated.)
