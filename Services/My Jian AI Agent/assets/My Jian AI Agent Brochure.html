How Jian Works
Four functions on a clock. One judgment loop.
Lightweight evaluators run on a fixed cadence; the AI gateway is pulled in only for judgment and context. Every output is a forensic, MITRE-tagged ticket routed to a real technician.
The Agent Functions — Cadence in Pacific Time
⚡
Alert
Every 15 min · lower tiers 30m / 1h / 2h / 6h
Real-time critical-event detection by fast evaluators; the LLM is pulled in only to add context.
Output: opens a P1/P2 Client-Portal ticket + a Teams channel + an Adaptive Card to "Jian Alerts."
🧠
Diagnose
Daily 14:00 PT · or at Alert cadence on critical platforms
The judgment pass — reads the day's data through the AI gateway and decides what warrants a ticket, the recommended fix, and the MITRE tag.
Output: a forensic CP ticket routed to the on-shift technician.
📊
Activity
Daily 14:30 PT
Daily roll-up of the day's events into a per-client security-posture summary.
Output: feeds the monthly reports, the SIEM correlator, and the learning loop.
🔢
Counts
Observed daily 05:00 PT · billed monthly
Usage and license counting — observed-vs-billed reconciliation that surfaces under-billing and license waste.
Output: read-only count reports for finance and ops.
Remediate — gated by design
