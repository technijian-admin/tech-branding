Chat.AI Tenant Onboarding

A short, one-time setup that connects your organization's Microsoft Entra ID (Azure AD) to Chat.AI, so your team can sign in securely with their existing Microsoft 365 accounts.

Chat.AI signs your people in with the Microsoft accounts they already use — no new passwords to manage. To make that connection, a Global Administrator in your Microsoft tenant registers a small application, creates one secret, grants a short list of permissions, and sends three values to Technijian. The whole process takes about 15–20 minutes, and you only do it once.

i Before You Begin
You'll need:
  • An account with the Global Administrator role in Microsoft Entra ID (formerly Azure Active Directory) — this lets you both register the app and grant admin consent in Step 9.
  • Access to the Azure portal at portal.azure.com.
  • A secure place to temporarily note three values (a password manager is ideal).
  • About 15–20 minutes. This is a one-time setup.

Chat.AI Tenant Onboarding
A Register Your Application & Create a Secret

First, create a dedicated application in your Microsoft tenant and give it one secret. This is the secure handshake that lets Chat.AI verify your users with Microsoft — Chat.AI never sees or stores their passwords.

1

Sign in to the Azure portal

Go to https://portal.azure.com and sign in with your Global Administrator account.

2

Open Microsoft Entra ID

In the search bar at the top, type Microsoft Entra ID and open it. (This was previously called Azure Active Directory.)

3

Create a new App registration

Select App registrations → New registration. Give it a clear name such as Technijian Chat.AI. Under Supported account types, choose Accounts in this organizational directory only (Single tenant). You can leave the Redirect URI blank. Click Register.

4

Copy and save two IDs

On the app's Overview page, copy and save both values — you'll send these to Technijian later:

•  Application (client) ID  •  Directory (tenant) ID

5

Create a client secret

Go to Certificates & secrets → Client secrets → New client secret. Add a description (e.g. Chat.AI), choose an expiry (we recommend 24 months), then click Add.

Important — copy the secret VALUE now. Copy the text in the Value column immediately and save it somewhere secure. This is not the Secret ID. The Value is shown only once — if you leave the page you cannot retrieve it and will have to create a new one.
You should now have three values saved: the Application (client) ID, the Directory (tenant) ID, and the Client Secret Value. Next you'll grant this app the permissions Chat.AI needs — continue on the next page.

B Grant the Required API Permissions
6

Open API permissions

In your app registration, select API permissions from the left menu, then click Add a permission.

7

Add Microsoft Graph permissions

Choose Microsoft Graph → Delegated permissions, then search for and add each of the four permissions below. Click Add permissions when done.

Microsoft Graph Delegated permissions
PermissionWhat it allows
offline_accessKeeps users signed in by allowing token refresh.
openidSigns users in using their Microsoft identity (OpenID Connect).
profileReads basic profile details (name, etc.) to personalize the app.
User.ReadReads the signed-in user's profile.
8

Add the Chat.AI API permissions

Click Add a permission again, open the APIs my organization uses tab, and search for TechChatV2 API — App 2. Choose Delegated permissions and add both below.

TechChatV2 API — App 2 Delegated permissions
PermissionWhat it allows
Chat.ReadRead access to your organization's Chat.AI conversations and data.
Chat.WriteCreate and send messages within your Chat.AI workspace.
Don't see TechChatV2 API — App 2 in the list? Contact your Technijian onboarding team — we'll provide the API's Application ID so you can search for it directly.
9

Grant admin consent

Back on the API permissions page, click Grant admin consent for <your organization> and confirm. All six permissions should then show a green ✓ Granted status.

That's everything on your side. Your app registration is now ready. Turn to the last page to send your three values to Technijian — we'll take it from there.

⚠ Keeping It Secure
  • Treat the client secret like a password. Don't paste it into a plain email or chat — ask your Technijian contact for our secure method to share it (or send the secret separately from the two IDs).
  • Secrets expire. We'll remind you before yours does. To renew, create a new secret (Step 5) and send us the new Value — access continues without interruption.
  • You stay in control. You can revoke Chat.AI's access at any time by deleting the secret or the app registration in Microsoft Entra ID.
